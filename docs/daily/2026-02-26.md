## 상세 작업 내역

### 1. 분석 프롬프트 보강 — 유사도 0% 방지 및 전 구간 일관성 (`8f631a8`)

- **문제:** 44페이지 중 22페이지부터 밑줄이 안 나오거나, 표절 문장인데 유사도가 0%로 추출되는 이슈 발생 (간헐적)
- **수정 — 프롬프트에 4가지 품질 지침 추가:**
  - `[유사도 0% 방지]` — 비교 카드가 존재하면 similarity 0%는 추출 오류로 간주. 근거 없는 0% 절대 금지
  - `[전 구간 일관성]` — 페이지 상단/중간/하단 비교 카드를 동일 강도로 전수 조사. 하단부 카드 건너뛰기 금지
  - `[하이라이트/밑줄 정밀 추출]` — 강조 부분만 정확히 분리, 색상과 수치의 논리적 일치 검증
  - `[AI 유사도 패턴 구체화]` — patternType에 "AI 패턴"이 아닌 구체적 근거 기술 강제
- **수정 — 후처리 보정 로직 추가:**
  - 비교 카드 존재 + `similarity === 0` + `highlightedText` 존재 → 최소값 1로 자동 보정
  - `originalText` 존재 + `highlightedText` 누락 → 콘솔 경고 출력

### 2. 고유사도(100%) 밑줄 누락 방지 (`c09b9ad`)

- **문제:** 유사도 100%인 카드에서 밑줄이 안 쳐지는 경우 발생
- **원인:** `originalText.includes(highlightedText)` 매칭 시, AI가 공백/줄바꿈을 약간 다르게 추출하면 `includes()`가 false 반환
- **수정 — 프롬프트 보강:**
  - 유사도 100%일 때 `highlightedText = originalText 전체`임을 명시
  - `highlightedText`는 반드시 `originalText`의 정확한 부분 문자열이어야 함을 강조
  - `highlightedText`가 빈 문자열이면 밑줄이 안 나온다는 점 경고
- **수정 — 코드 3단계 fallback:**
  1. 정확한 `includes()` 매칭 (기존)
  2. 공백 정규화(`\s+` → 단일 공백) 후 재매칭
  3. 유사도 80% 이상 + 매칭 완전 실패 → 전체 밑줄 처리
- **수정 — 후처리 자동 보정:**
  - 유사도 80% 이상인데 `highlightedText` 누락 → `originalText` 전체로 자동 채움

### 3. 무의미한 비교 카드 필터링 (`b5802a8`)

- **문제:** 비교문서정보 표의 행이 비교 카드로 잘못 추출되는 경우 (원문에 번호만 표시)
- **수정 — 프롬프트 규칙 7번 추가:**
  - 원문이 숫자만 있거나(27, 28 등) 의미 있는 문장이 아닌 카드는 제외
- **수정 — 후처리 필터링:**
  - `originalText`가 숫자만(`/^\d+$/`) → 자동 제거
  - `originalText`가 2글자 이하 → 자동 제거

### 4. 비교문서정보 표 설명 실제 구조 반영 (`d518e6f`)

- **문제:** 프롬프트의 비교문서정보 표 설명이 실제 PDF 구조와 불일치 (검사문서/비교문서 2열 → 실제는 번호/표절률/출처정보 3열)
- **수정:**
  - 표 구조를 `번호 | 표절률 | 출처정보` 3열로 정확하게 기술
  - 여러 페이지에 걸칠 수 있으며, 첫 페이지에만 헤더가 존재함을 명시
  - 헤더 없는 후속 페이지도 동일하게 무시하도록 보강

### 5. 중복 비교 카드 제거 (`76d9df5`)

- **문제:** 동일한 비교 카드가 결과에 2개씩 중복으로 나타남
- **수정:**
  - 전체 페이지 결과 합산 후 중복 제거 로직 추가
  - 판별 키: `originalText 앞 80자 + similarity + pageNum` 조합
  - 중복 발견 시 첫 번째만 유지, 콘솔에 제거 로그 출력
