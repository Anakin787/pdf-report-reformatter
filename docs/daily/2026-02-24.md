## 상세 작업 내역

### 1. PDF 다운로드 파일명 형식 변경 (`d7ccd10`)

- **이전:** 원본 PDF 파일명 그대로 저장 (예: `프로포절 발표문.pdf`)
- **이후:** `카피클린 유사도 분석 결과 리포트 - 프로포절 발표문.pdf` 형식으로 변경
- 자동 다운로드, 수동 다운로드 버튼 모두 적용

### 2. 문서명 추출 정확도 개선 (`d7ccd10`, `060f313`)

- **문제:** 공백 포함 파일명 (예: "프로포절 발표문.pdf")이 "발표문.pdf"로 잘려서 추출됨
- **원인 1:** 텍스트 정규식이 `[^\s]` (공백 제외)로 매칭
- **원인 2:** `.+?` 패턴이 PDF 텍스트 전체를 캡처하여 파일명에 긴 텍스트가 들어감
- **수정:**
  - 정규식을 `[\w가-힣\s\-_.()]{1,80}` 로 변경 (파일명 문자셋 + 80자 제한)
  - "문서명" 라벨 패턴 추가 (카피킬러 PDF 호환)
  - AI 프롬프트에 파일명 축약 금지 가이드 추가
  - AI 결과와 텍스트 fallback 중 더 긴(완전한) 파일명 선택 로직

### 3. 차트 초과 chip 잘림 수정 (`d7ccd10`)

- **문제:** 표절률이 높을 때 "+59% 초과" chip이 차트 canvas 상단에서 잘림
- **수정:**
  - chip y 위치를 `Math.max(4, yPos - 52)`로 클램핑
  - y축 max 여유 공간 `+15` → `+25`로 확대
  - 퍼센트 텍스트가 chip과 겹치지 않도록 위치 보정

### 4. Gemini API 배열 응답 처리 (`93b6e05`)

- **문제:** Gemini가 `[{...}]` 배열로 반환하는데 코드는 `{...}` 객체를 기대하여 1페이지 데이터(표절률, 문장 통계 등)가 전부 0으로 표시됨
- **수정:** `extractMetadataFromPDF`, `analyzeWithGemini` 모두 `Array.isArray(parsed) ? parsed[0] : parsed` 처리

### 5. STEP 3.5/4 에러 방어 로직 강화 (`1db2e5f`)

- **문제:** 정밀 분석(STEP 3.5) 중 API 에러 발생 시 전체 processFile이 catch로 빠져 뷰 활성화(`pages=[1,2]`)가 실행되지 않아 1페이지가 표시되지 않음
- **수정:**
  - STEP 3.5 루프 내 페이지별 try-catch 추가 (에러 시 해당 페이지만 건너뛰고 계속 진행)
  - STEP 4 진단 총평 생성에 try-catch 추가 (실패 시 기본값 사용)
  - 텍스트 fallback에서 문장 통계(유사/동일/의심/인용) 정규식 추출 추가

### 6. 비교카드 높이 유동 계산 (`da718f0`, `288ab88`, `4cddf7d`)

- **이전:** 모든 카드를 고정 240px로 계산하여 페이지 분배 → 긴 텍스트 카드가 넘침
- **이후:** 텍스트 길이 기반 높이 추정 (최소 190px 보장)
  - 좌측(원문)/우측(비교문) 중 더 긴 쪽의 줄 수 계산
  - `줄 수 × 줄 높이 + 헤더 + 푸터 + 패딩 + 마진`으로 카드 높이 산출
  - `CHARS_PER_LINE` 28→42로 실측 보정, 한 페이지당 카드 4개→5개로 증가

### 7. 다운로드 파일명 .pdf.pdf 중복 방지 (`cfeb43b`)

- **문제:** 원본 문서명이 "프로포절 발표문.pdf"이고 autoSavePDF에서 `.pdf`를 붙여 `.pdf.pdf`가 됨
- **수정:** docName에서 원본 확장자(`.pdf`, `.hwp`, `.docx` 등)를 제거 후 saveName 생성

### 8. PDF 다운로드 방식 변경 — jsPDF → 브라우저 인쇄 (`95e4613`)

- **이전:** jsPDF + html2canvas로 자동 PDF 생성 → 깨짐 발생
- **이후:** `window.print()` 기반 브라우저 인쇄 창에서 사용자가 직접 "PDF로 저장"
- jsPDF, html2canvas 스크립트 및 `autoSavePDF` 함수 제거
- `document.title`에 파일명 설정하여 인쇄 시 기본 파일명으로 사용

### 9. reportId 고정 — CC-랜덤8자리 (`cd1cec0`)

- **문제:** AI/텍스트 추출값이 reportId를 덮어쓰는 경우 발생
- **수정:** reportId는 항상 `CC-랜덤8자리`로만 설정, 추출된 검사번호는 `inspectionNo`에만 반영

### 10. 다중 파일 처리 시 파일별 개별 다운로드 (`396275f`)

- **문제:** 다중 파일 순차 처리 시 마지막 파일 결과만 남아 이전 파일 다운로드 불가
- **수정:**
  - `waitingForDownload` 상태 추가, `startQueue`에서 파일별 처리 후 대기
  - "📥 다운로드 & 다음 파일 →" 버튼으로 현재 결과 인쇄 후 다음 파일 진행
  - 마지막 파일은 기존 "📥 리포트 다운로드" + "초기화" 버튼 표시

### 11. 페이지 분석 수 제한 해제 (`b73d3b2`)

- **이전:** `Math.min(pdf.numPages, 10~15)`로 분석 페이지 수 제한
- **이후:** `pdf.numPages` 전체 페이지 분석
